# ClipWiz 项目代理配置

## 项目概述

ClipWiz 是一个视频编辑与合成平台，采用 **pnpm workspace monorepo** 架构。
用户在前端进行多轨道视频编辑，实时通过 WebGL（VideoContext）预览，最终由后端 FFmpeg 完成合成导出。

```
clipWiz/
├── apps/web           → 前端编辑器（React + VideoContext WebGL 预览）
├── apps/server        → 后端合成服务（Express + FFmpeg）
├── packages/shared    → 前后端共享类型定义 & 工具函数
└── packages/videoPlayer → VideoContext 封装的播放引擎
```

### 包依赖关系

```
apps/web ──────────────────► @clipwiz/shared
    └──────────────────────► @clipwiz/videoPlayer ──► @clipwiz/shared
apps/server ────────────────► @clipwiz/shared
```

## 可用代理及职责分工

### Frontend Agent
**触发条件**: `apps/web/` 下的所有代码，包括 React 组件、UI、Vite 配置、Less 样式、EventBus 通信
**技术栈**: React 18 + TypeScript + Vite + Ant Design 5 + Less
**负责模块**:
- `App.tsx` — 路由入口（`/` 项目首页，`/editor/:id` 编辑页）
- `pages/HomePage/` — 项目列表与新建/重命名/删除
- `pages/EditorPage/` — 编辑器页面（Header / LeftCon / VideoPlayer / TimeLine / MaterialEditor）
- `components/leftCon/` — 左侧素材面板（`MaterialList` / `AudioList` / `PhotoList` 等 Tabs）
- `components/timeLine/` — 多轨道时间轴（基于 `@xzdarcy/react-timeline-editor`）
- `components/videoPlayer/` — 画布预览区（`<canvas>` + `VideoControls` 控制栏）
- `store/editorStore.ts` — 编辑状态中心（Zustand）
- `utils/eventBus.ts` — 跨组件通信（`video:play` / `video:pause` / `video:seek` / `time:update`）
- `utils/request.ts` — fetch 封装，统一调用后端 API
- `mock.ts` — 前端本地测试用 `ITrackInfo` 数据

**不负责**: packages/videoPlayer 内部逻辑、后端 API 实现

---

### Backend Agent
**触发条件**: `apps/server/` 下的所有代码，包括路由、服务、队列、中间件
**技术栈**: Node.js ≥18 + TypeScript (tsx) + Express 4 + fluent-ffmpeg + Bull + ioredis + Multer + Mongoose + Ali-OSS
**负责模块**:
- `routes/upload.ts` — 文件上传（单个/批量，Multer，UUID 命名，500MB 限制）
- `routes/video.ts` — 视频操作（裁剪 `trim` / 合并 `merge` / 水印 `watermark` / 转码 `transcode`）
- `routes/job.ts` — Bull 任务管理（查询状态/进度/取消）
- `routes/graph.ts` — **核心合成路由**：接收 `ITrackInfo` → 调用 `videoProcessor` → 返回下载链接
- `routes/project.ts` — 项目 CRUD（协议持久化）
- `routes/material.ts` — 视频/图片素材上传（临时本地 -> OSS -> Mongo）
- `routes/audio.ts` — 音频列表/上传/删除/预设入库
- `routes/filter.ts` — 滤镜列表/预设入库
- `routes/transition.ts` — 转场列表/预设入库
- `services/videoProcessor.ts` — FFmpeg 合成逻辑，`generateCompositeCommand()`
- `services/queue.ts` — Bull 队列初始化与任务处理器
- `services/ossService.ts` — OSS 上传/删除
- `services/mock.ts` — 服务端测试用 `ITrackInfo` 数据
- `middleware/errorHandler.ts` — 全局错误处理中间件

### Video Processing Agent
**触发条件**: 涉及 FFmpeg 命令生成、视频编解码参数、VideoContext 渲染逻辑、PAG 动画
**负责模块**:
- `apps/server/src/services/videoProcessor.ts` — FFmpeg 合成命令构建
- `packages/videoPlayer/src/components/video.ts` — VideoContext 视频节点
- `packages/videoPlayer/src/components/audio.ts` — 音频节点（BGM / 口播）
- `packages/videoPlayer/src/components/photo.ts` — 图片 / GIF 帧解码节点
- `packages/videoPlayer/src/components/subtitle.ts` — PAG 字幕节点
- `packages/videoPlayer/src/components/text.ts` — PAG 文字贴图节点
- `packages/videoPlayer/src/components/filter.ts` — 滤镜特效节点
- `packages/videoPlayer/src/components/Pag.ts` — libpag 初始化与 Canvas 渲染

**已知问题（修改时注意）**:
- `filter.ts`: 目前仅接入 `MONOCHROME` 预览效果，`item.code`/真实滤镜映射尚未落地
- `subtitle.ts`: `addSubtitleNode` 中字幕文本 **hardcode**，需改为从轨道数据读取
- `videoProcessor.ts`: 当前后端导出仍以视频+音频+转场为主，字幕/花字/贴图/滤镜未完整对齐到导出侧

### Shared Types Agent
**触发条件**: 涉及 `packages/shared/` 的类型定义、API 接口、枚举、常量、工具函数
**技术栈**: 纯 TypeScript，无运行时依赖，编译为 ESM
**负责模块**:

`packages/shared/src/video.ts` — **核心数据模型**（最复杂，前后端共用）:
```typescript
ITrackInfo     → 合成任务顶层结构 { width, height, duration(ms), tracks[] }
ITrack         → 单条轨道 { type: MATERIAL_TYPE, items[] }
IVideoTrackItem → 视频轨道条目（含变速/裁剪/转场/transform）
IPhotoTrackItem → 图片贴图条目
ITextTrackItem  → PAG 文字条目
ISubtitleItem   → PAG 字幕条目
IAudioTrackItem → 音频条目（BGM/口播）
IFilterTrackItem→ 滤镜条目
```

`packages/shared/src/enum.ts`:
```typescript
MATERIAL_TYPE: VIDEO | BGM_AUDIO | ORAL_AUDIO | SUBTITLE | TEXT | PHOTO | FILTER | BACKGROUND
```

`packages/shared/src/constants.ts` — API 路径常量、分辨率预设（`RESOLUTION_PRESETS`）、支持格式列表

`packages/shared/src/types.ts` — 通用 API 响应类型（`ApiResponse<T>`, `JobInfo`, `TimelineItem`）

`packages/shared/src/utils.ts` — `formatDuration` 等格式化工具

**约定**: 所有跨包共用的类型 **只能** 在此包中定义，禁止在 apps/ 中重复声明相同结构。

## 核心数据流

### 前端预览流程

```
ITrackInfo (mock.ts 或后端返回)
    │
    ▼
packages/videoPlayer › Editor.loadTrack()
    ├── VIDEO        → videoCtx.video(url) + transition registry/manageTransitions
    ├── BGM_AUDIO    → videoCtx.audio(url)
    ├── ORAL_AUDIO   → videoCtx.audio(url)
    ├── PHOTO        → videoCtx.image(url) / GIF 逐帧解码
    ├── SUBTITLE     → PAG Canvas → videoCtx.canvas()
    ├── TEXT         → PAG Canvas → videoCtx.canvas()
    └── FILTER       → videoCtx.effect(MONOCHROME)
    │
    ▼
apps/web › VideoPlayer <canvas>
    └── requestAnimationFrame → Editor.draw() → WebGL 合成帧
```

### 后端导出流程

```
前端点击"导出视频"
    │ POST /api/graph  { trackInfo: ITrackInfo }
    ▼
apps/server › routes/graph.ts
    │
    ▼
services/videoProcessor.ts › generateCompositeCommand()
    │  fluent-ffmpeg 构建 FFmpeg 过滤器图
    ▼
uploads/output/<uuid>.mp4
    │ 返回 { downloadUrl }
    ▼
前端 utils/downloadFile.ts 触发下载
```

### 跨组件通信（EventBus）

| 事件 | 触发方 | 监听方 | 说明 |
|------|--------|--------|------|
| `video:play` | VideoPlayer | TimeLine | 时间轴开始跟随 |
| `video:pause` | VideoPlayer | TimeLine | 时间轴暂停 |
| `video:seek` | VideoControls | TimeLine | 同步时间轴游标 |
| `time:update` | TimeLine (afterSetTime) | VideoPlayer | 拖动时间轴 → 播放器 seek |

## 项目规范

### TypeScript 规范
- 所有跨包共用类型只在 `packages/shared` 中定义
- 禁止使用 `any`，优先用 `unknown` + 类型守卫
- 时间单位统一使用 **毫秒（ms）**，VideoContext API 传入时除以 `MILL_TIME_CONVERSION(1000)` 转换为秒
- 帧时长常量：`FRAME_TIME = 0.04s`（25fps），帧步进使用 `±FRAME_TIME`

### 代码风格
- 前端使用 React 函数组件 + Hooks，禁止 Class 组件
- 后端使用 `async/await`，所有 async 路由必须被 `try/catch` 包裹
- 样式使用 Less（非 CSS Modules），类名遵循 BEM 风格
- 组件目录下同名 `index.tsx` + `index.less`

### 文件组织
- 新增 API 路由 → 在 `apps/server/src/routes/` 新建文件，并在 `index.ts` 挂载
- 新增轨道类型 → 先在 `packages/shared/src/enum.ts` 添加枚举，再在 `video.ts` 补充接口，最后分别在 `videoPlayer` 和 `videoProcessor` 实现
- 新增前端组件 → 放在 `apps/web/src/components/` 对应模块目录下

### 安全规范
- 文件下载路由必须做 **path traversal 防护**（参考 `routes/graph.ts` 已有实现）
- 上传文件使用 UUID 重命名，禁止直接使用用户传入的文件名
- 跨域配置使用白名单，不允许 `origin: *` 用于生产环境

### 状态管理约定
- 当前主编辑状态使用 Zustand（`store/editorStore.ts`）
- 组件局部状态继续使用 `useState`
- 跨层级/跨模块通信用 `EventBus`（`utils/eventBus.ts`）
- **不要**引入 Redux；如需全局状态扩展，优先使用已安装的 Zustand

## 开发流程

### 常用命令

```bash
# 安装依赖
pnpm install

# 启动全部服务（并行）
pnpm dev

# 单独启动
pnpm dev:web      # 前端 → http://localhost:3000
pnpm dev:server   # 后端 → http://localhost:4000

# 构建生产版本
pnpm build

# 清理构建产物
pnpm clean

# 检查环境依赖（Node.js / pnpm / FFmpeg / Redis）
bash scripts/check-env.sh
```

### 环境依赖
| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| Node.js | ≥ 18 | 运行时 |
| pnpm | ≥ 8 | 包管理 |
| FFmpeg | 系统安装 | 视频处理 |
| Redis | 可选 | Bull 任务队列（lazyConnect，不启动也可运行）|
| MongoDB | 推荐本地启动 | 项目与素材元数据存储 |
| 阿里云 OSS 凭证 | 上传素材时必需 | 素材云存储 |

### 文件存储路径
```
apps/server/uploads/
├── <uuid>.<ext>        ← 上传的原始素材
└── output/
    └── <uuid>.mp4      ← 合成输出文件
```
